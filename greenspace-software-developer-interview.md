# Software Design & Development Interview
## Candidate Preparation Guide

**Greenspace Health**

Hi! We're excited to have you join us for the Software Design & Development interview. This guide will help you understand what to expect and how to prepare.

## What to Expect

This interview is designed to simulate how we actually work together as a team at Greenspace. Think of it as a collaborative working session, not an exam. You'll pair with our team on a real coding task using the tools and workflows you use every day — including AI coding agents.

| | |
|---|---|
| **Duration** | 90 minutes (scheduled), typically wraps within 75 minutes |
| **Format** | Architecture discussion → AI-assisted live coding → Code review |
| **Interviewers** | Mike (CTO), Cole (Senior Software Developer), and one developer from the team you'd be joining |
| **Platform** | Zoom video call with screenshare |

## Choose Your Track

You have two options for the coding portion. Choose whichever you feel will best showcase your abilities. In both cases, the application should be a Typeform-like form builder application. Please let us know your choice at least 3 days before the interview.

### Option A: Use Our Playground Repository

We'll provide you with a GitHub repository containing a basic web application (a Typeform-style form builder). You'll familiarize yourself with it beforehand, then during the interview we'll discuss the architecture and pair on adding a feature together.

**What to prepare:**

- Clone the repo (link provided separately) and get it running locally
- Explore the codebase — understand the tech stack, data model, and how the frontend and backend connect
- Form opinions about what you'd improve or do differently
- Estimated prep time: 1–2 hours

### Option B: Build Your Own Form Builder Application

Build your own version of a Typeform-style form builder ahead of the interview using any tech stack you're comfortable with. During the session, you'll walk us through what you built, discuss your design decisions, and then we'll pair on extending it together.

**Requirements for your app:**

- A working Typeform-style form builder with a frontend, backend/API, and data persistence (database)
- At minimum: ability to create forms with different field types, collect responses, and view results
- Any tech stack you're comfortable with — we're stack-agnostic
- Hosted on GitHub (or similar) so we can review the code
- Runnable locally with a clear setup (README with instructions)
- Scope it appropriately — this doesn't need to be production-ready. We're more interested in your architectural choices and design decisions than completeness
- You're encouraged to use AI tools (Claude Code, Cursor, Copilot, etc.) to build it! We'll want to hear about your process.
- Share the repo link at least 24 hours before the interview so our team can review it
- Estimated prep time: 2–4 hours (including building and polishing)

## Tooling & Setup

Please have the following ready on the day of the interview:

- Your preferred IDE (VSCode, Cursor, JetBrains, etc.) with the application running locally
- Your preferred AI coding agent (Claude Code, GitHub Copilot, Cursor AI, OpenCode, Codex, AugmentCode — whatever you use day-to-day). We want to see your real workflow.
- Zoom installed with the ability to screenshare your IDE and terminal
- (Optional) VSCode Live Share if you'd like to enable our interviewers to navigate the code alongside you. Not required.

## What We're Looking For

This isn't about solving tricky algorithmic puzzles or knowing obscure trivia. Here's what we're interested in:

1. **How you think about software design** — your approach to architecture, data modeling, API design, and trade-offs.
2. **How you work with AI tools** — your prompting strategy, how you review AI output, and when you choose to code something yourself.
3. **How you collaborate** — think aloud, ask questions, and engage with our team. We're looking for a great teammate.
4. **Your fundamentals** — solid understanding of web architecture, the stack you're working in, and good engineering practices.
5. **Your quality instincts** — how you think about testing, edge cases, code maintainability, and what "done" means.

## Tips for Success

- **Use your real workflow.** Don't try to impress us by avoiding AI or by over-relying on it. Use the tools the way you normally would.
- **Think aloud.** The most important thing is hearing your reasoning. Narrate your thought process, even when you're prompting an AI tool.
- **It's okay to not finish.** We'd rather see a thoughtful partial implementation than a rushed complete one. Prioritize well.
- **Ask clarifying questions.** In a real working environment, requirements are ambiguous. We want to see how you handle that.
- **Be yourself.** We're trying to understand what it's like to work with you. Relax, be natural, and have fun with it.

We're looking forward to building something together. If you have any questions about the format or preparation, don't hesitate to reach out to your recruiter.
